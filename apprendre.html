Comment fonctionne le lien entre CSS en haut et HTML en bas ? Par exemple .info-card h3 et mon <h3> dans le code ?

C'est ce qu'on appelle un sélecteur CSS. Il dit : "Applique ce style à tous les h3 qui se trouvent DANS un élément avec la classe info-card".

/* CSS en haut */ .info-card h3 { color: var(--vert2); font-size: 16px; } /* HTML en bas — le navigateur fait le lien automatiquement */ <div class="info-card"> <h3>💡 Mon titre</h3> ← reçoit le style </div>
Astuce : Le CSS s'applique automatiquement à tous les éléments qui correspondent au sélecteur — pas besoin de répéter le style à chaque fois !

Les balises HTML — comment ça fonctionne ? Ouvrir et fermer ?

La règle est toujours la même : ouvrir puis fermer avec un slash /

<balise>contenu</balise>
BaliseEffetExemple
<strong>Gras<strong>texte</strong>
<em>Italique<em>texte</em>
<p>Paragraphe<p>texte</p>
<h1> à <h6>Titres<h2>Mon titre</h2>
<a>Lien<a href="url">texte</a>
<img>Image<img src="photo.jpg">
<br>Saut de lignepas de fermeture
Erreur classique : </strong>texte</strong> ← le slash ne doit pas être au début ! La bonne version : <strong>texte</strong>

C'est quoi la différence entre <br><br> et des <p> séparés ?

/* Mauvaise pratique — simuler des paragraphes avec br */ <p>Premier texte.<br><br> Deuxième texte.</p> /* Bonne pratique — vrais paragraphes séparés */ <p>Premier texte.</p> <p>Deuxième texte.</p> /* Gérer l'espace dans le CSS */ p { margin-bottom: 1rem; }

Les div imbriquées — comment ne pas se perdre dans les fermetures ?

Chaque <div> ouverte doit être fermée avec </div>. Entre les vidéos on a 2 fermetures, à la toute dernière on en a 3 :

<div class="vid-card"> <div class="vid-embed"> <iframe src="..."></iframe> </div> ← ferme vid-embed <div class="vid-info"> <div class="vid-title">Titre</div> <div class="vid-date">Date</div> </div> ← ferme vid-info </div> ← ferme vid-card /* Dernière vidéo : ajouter encore un </div> pour fermer le conteneur videos */ </div> ← ferme videos (seulement à la fin !)
Astuce pro : Ajouter des commentaires pour s'y retrouver !
</div><!-- fin vid-card --> — invisible sur la page mais très utile !
🎨 CSS — Le Style
❓

C'est quoi la différence entre le style inline et le CSS en haut ?

/* Style inline — directement dans la balise HTML */ <p style="font-size:13px;color:red">texte</p> /* CSS en haut — bien mieux ! */ .info-card p { font-size: 13px; color: red; } /* Et dans le HTML simplement : */ <p>texte</p> ← plus besoin du style inline !
Avantage CSS en haut : Si tu veux changer la couleur un jour, tu ne la changes qu'à un seul endroit et ça se met à jour partout automatiquement !
❓

Comment mettre un mot en gras ET en vert avec <strong> ?

/* Option 1 — style inline (rapide mais pas propre) */ <strong style="color:#6aad5e">texte</strong> /* Option 2 — dans le CSS (propre et réutilisable) */ .info-card strong { color: var(--vert2); } .hero-info strong { color: var(--vert2); } /* Option 3 — partout sur la page */ strong { color: var(--vert2); }
❓

C'est quoi les variables CSS comme var(--vert2) ?

Les variables CSS se définissent une fois en haut dans :root et s'utilisent partout avec var(--nom) :

:root { --vert: #4a7c3f; --vert2: #6aad5e; ← vert clair --or: #c8941a; } /* Utilisation */ h3 { color: var(--vert2); } strong { color: var(--vert2); }
Avantage : Si tu veux changer le vert de tout ton site, tu changes juste une ligne dans :root !
❓

C'est quoi le media query et comment l'utiliser pour le responsive mobile ?

Le media query applique des styles différents selon la taille de l'écran :

/* Style normal — PC */ .hero { grid-template-columns: 1fr 1fr; } ← 2 colonnes /* Style mobile — écran < 768px */ @media(max-width:768px) { .hero { grid-template-columns: 1fr; } ← 1 colonne .hero-img img { height: 220px; } ← image moins haute .videos { grid-template-columns: 1fr; } }

Comment utiliser VS Code directement dans GitHub sans rien installer ?

Sur n'importe quelle page de ton repo GitHub, appuie sur la touche . (point) du clavier.

VS Code s'ouvre dans le navigateur avec :

✓ Coloration syntaxique — chaque balise a sa couleur ✓ Div ouvrante/fermante surlignée au clic ✓ Erreurs en rouge automatiquement ✓ Ctrl+S pour sauvegarder → commit GitHub
Enregistrer : Ctrl+S → petit message de commit → Commit & Push

Comment ajouter des emojis dans le HTML ?

Les emojis fonctionnent directement dans le HTML — juste copier-coller !

<h3>🥚 Premier œuf détecté !</h3> <p>🐣 Éclosion aujourd'hui !</p> <span>📅 24 mars 2026</span>

Sites pour trouver des emojis :

emojipedia.org — chercher en anglais (egg, bird, nest...) getemoji.com — parcourir par catégories

Les commentaires HTML — comment ça marche ?

Un commentaire est invisible sur la page mais très utile pour s'y retrouver dans le code :

<!-- Ceci est un commentaire, invisible sur la page --> </div><!-- fin vid-card --> </div><!-- fin vid-info --> </div><!-- fin videos -->
Astuce : Utilise les commentaires pour marquer tes fermetures de div — tu ne te perdras plus jamais !
